# Governance

gpu-mcp-server follows the [Linux Foundation Minimum Viable Governance](https://github.com/github/MVG) model.

## Maintainers

Maintainers have write access to the repository and are responsible for reviewing
and merging contributions, managing releases, and guiding the project roadmap.

New maintainers are added by unanimous agreement of existing maintainers after
sustained contribution (typically 3+ months of active involvement).

## Decision Making

- **Consensus preferred.** Most decisions are made through discussion on GitHub
  issues and pull requests.
- **Voting.** When consensus cannot be reached, maintainers vote. A simple
  majority of active maintainers is required.
- **Lazy consensus.** For routine changes (bug fixes, documentation, dependency
  updates), a PR can be merged after one maintainer approval and 24 hours without
  objection.

## Contributions

All contributions follow the process described in [CONTRIBUTING.md](CONTRIBUTING.md).
Contributors must sign off on commits (DCO) to certify they have the right to
submit the code under the project's license.

## Code of Conduct

This project follows the [Contributor Covenant v2.1](CODE_OF_CONDUCT.md).

## Changes to Governance

Changes to this document require approval from all active maintainers.
